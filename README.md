# VAS Banking ATM Application

A Python-based banking ATM system with comprehensive test coverage. This project implements core banking operations including login, deposits, withdrawals, transfers, and bill payments with various account types and validation rules.

## Project Structure

```
.
├── bank_atm.py                 # Main Python ATM application
├── README.md                   # This file
├── source_code/                # Web interface components
│   ├── app.js                  # JavaScript application logic
│   ├── styles.css              # CSS styling
│   └── *.html                  # HTML pages (login, dashboard, deposit, etc.)
└── tests/                      # Test suite
    ├── currentaccounts.txt     # Test account definitions
    ├── failure_table.csv       # Failure tracking in CSV format
    ├── failure_table.md        # Failure tracking in Markdown format
    ├── expected/               # Expected output files (TC01.atf - TC84.atf)
    ├── inputs/                 # Test input files (TC01.in - TC84.in)
    ├── outputs/                # Generated test output files
    └── scripts/
        ├── run_all.ps1         # Run all 84 tests
        ├── check_out.ps1       # Check test output files
        ├── check_atf.ps1       # Check transaction files
        ├── bless_expected.ps1  # Bless expected outputs
        └── make_failure_table.ps1  # Generate failure tracking table
```

## Running the Tests

### Prerequisites
- Python 3.x installed
- PowerShell (Windows) or compatible shell

### Run All Tests
Execute all 84 test cases and generate output files:

```powershell
.\tests\scripts\run_all.ps1
```

This will:
1. Read each test input file from `tests/inputs/TC*.in`
2. Run it through the ATM application
3. Generate `.atf` (transaction file) and `.out` (terminal output) files in `tests/outputs/`

### Run a Single Test
To run an individual test case (example: TC01):

```powershell
Get-Content tests/inputs/TC01.in | python bank_atm.py tests/currentaccounts.txt tests/outputs/TC01.atf | Set-Content tests/outputs/TC01.out
```

Or from a bash-compatible shell:
```bash
python bank_atm.py tests/currentaccounts.txt tests/outputs/TC01.atf < tests/inputs/TC01.in > tests/outputs/TC01.out
```

### Verify Test Results

Check test output files against expected outputs:
```powershell
.\tests\scripts\check_out.ps1   # Check terminal output files
.\tests\scripts\check_atf.ps1   # Check transaction files
```

Generate or update the failure tracking table:
```powershell
.\tests\scripts\make_failure_table.ps1
```

## Test Files

- **TC01 - TC84**: 84 test cases covering various banking operations
- **inputs/**: Input commands for each test (what the user types)
- **expected/**: Expected transaction files (.atf) for each test
- **outputs/**: Generated files after running tests
  - `.atf` files: Transaction log generated by the application
  - `.out` files: Terminal output (user-visible messages)

## Account Management

Test accounts are defined in `tests/currentaccounts.txt` and include various account types with different permission levels and limits (standard accounts, admin accounts, etc.)
